    Replace the placeholder in `src/app/settings/page.tsx` with a full client component.

    1. Add `'use client'` directive at the top (required — IndexedDB is browser-only, per research pitfall 4).

    2. Import React hooks, Link, getProfile, saveProfile, UserProfile.

    3. Component state:
       - `profile: UserProfile | null` — loaded from IndexedDB on mount
       - `loading: boolean` — true until first getProfile() resolves
       - Three input states: `allergyInput`, `preferenceInput`, `dislikeInput`

    4. Load profile on mount in useEffect:
       ```typescript
       useEffect(() => {
         getProfile().then((p) => {
           setProfile(p ?? { allergies: [], preferences: [], dislikes: [] });
           setLoading(false);
         });
       }, []);
       ```

    5. Helper function `addItem(field: 'allergies' | 'preferences' | 'dislikes', value: string)`:
       - Trim and lowercase the value
       - Guard: return early if empty or already in the list
       - Create updated profile with new item added
       - Call `saveProfile(updated)` then `setProfile(updated)`

    6. Helper function `removeItem(field: 'allergies' | 'preferences' | 'dislikes', value: string)`:
       - Filter the value out of the array
       - Call `saveProfile(updated)` then `setProfile(updated)`

    7. UI structure using Tailwind (accessible, high-contrast per CLAUDE.md):
       ```
       <div className="max-w-lg mx-auto px-4 py-6">
         <h1>Settings</h1>

         {loading ? <p>Loading...</p> : (
           <>
             <ProfileSection
               title="Allergies"
               description="Items flagged as safety-critical in conversation"
               items={profile.allergies}
               inputValue={allergyInput}
               onInputChange={setAllergyInput}
               onAdd={() => addItem('allergies', allergyInput).then(() => setAllergyInput(''))}
               onRemove={(item) => removeItem('allergies', item)}
               inputLabel="Add allergy"
               inputPlaceholder="e.g., peanuts, dairy, shellfish"
             />

             <ProfileSection title="Dislikes" ... />
             <ProfileSection title="Preferences" ... />
           </>
         )}

         <Link href="/">Back to Home</Link>
       </div>
       ```

    8. ProfileSection can be an inline component or extracted helper. Each section renders:
       - Section heading (h2) with title
       - Description text (p, text-gray-600)
       - List of current items, each with:
         - Item text
         - Remove button with `aria-label="Remove {item}"` (accessible)
         - Remove button uses min-h-[48px] touch target per CLAUDE.md A11Y-03
       - Form with text input + "Add" button
         - Input has `aria-label="{inputLabel}"`
         - Form `onSubmit` calls addItem and clears input
         - Add button has min-h-[48px] touch target
       - Empty state text when list is empty: "No {title.toLowerCase()} added yet"

    9. Styling guidelines (consistent with existing app):
       - `focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-black` on all interactive elements
       - `min-h-[48px]` on buttons for touch targets
       - `text-lg` on list items for readability
       - Remove buttons: `bg-red-600 text-white rounded px-3 py-1` with hover state
       - Add button: `bg-black text-white rounded-lg px-4 py-2 font-medium`
       - Section spacing: `mb-8` between sections
