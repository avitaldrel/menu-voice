<objective>
Extend the app state machine with the retake variant needed by the guided retake flow, create the AppStateAnnouncer component for ARIA announcements of app state transitions, and add page titles for the Next.js route announcer.

Purpose: A11Y-02 requires all app state changes to be announced via ARIA live regions. The existing ProcessingState and VoiceStateIndicator cover voice loop states, but extraction completion (processing -> results) has no announcement. The settings page also lacks a distinct page title for the route announcer. Additionally, the retake app state must exist before Plan 02 can build the guided retake flow.

Output: Extended AppState type, AppStateAnnouncer component, settings layout with metadata, updated main page title, and comprehensive tests.
