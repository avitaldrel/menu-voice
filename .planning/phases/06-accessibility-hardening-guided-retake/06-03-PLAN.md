Add welcome TTS message on first user interaction and verify the complete VoiceOver end-to-end flow works without sighted assistance.

Purpose: A11Y-01 requires all functionality to be accessible via voice alone. The existing sr-only h1 is announced by VoiceOver when focus lands on it, but adding a TTS welcome on the first user interaction reinforces the message and provides audio confirmation the app is working. iOS Safari blocks audio playback before a user gesture, so the welcome TTS must be chained to the first ScanButton tap (the first interactive gesture in idle state), not fired on page mount. After this plan, the app satisfies A11Y-01: a blind user can independently complete the full flow.

    Modify src/hooks/useVoiceLoop.ts:

    1. Add a ref to track whether welcome has been played:
       ```typescript
       const hasPlayedWelcomeRef = useRef(false);
       ```

    2. Add the `speakWelcome` callback after the `ensureInstances` definition:
       ```typescript
       const speakWelcome = useCallback(() => {
         if (hasPlayedWelcomeRef.current) return;
         hasPlayedWelcomeRef.current = true;
         ensureInstances();
         const welcomeText = 'Welcome to MenuVoice. Tap Scan Menu to photograph a restaurant menu.';
         ttsClientRef.current?.queueText(welcomeText);
         ttsClientRef.current?.flush();
         dispatch({ type: 'SPEECH_RESULT', transcript: '' });
       }, [ensureInstances]);
       ```

       Design decisions:
       - Called from ScanButton tap handler (user gesture) — satisfies iOS Safari autoplay policy per RESEARCH.md
       - Uses SPEECH_RESULT with empty transcript to enter processing -> speaking flow via existing state machine
       - One-shot via ref guard — never repeats
       - Uses TTSClient (audio element) per CLAUDE.md — NOT SpeechSynthesis
