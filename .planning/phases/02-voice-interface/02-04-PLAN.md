  <action>
    Create `src/hooks/useVoiceLoop.ts`:

    ```typescript
    'use client';
    import { useReducer, useRef, useCallback, useEffect, useState } from 'react';
    import { voiceReducer, initialVoiceState, type VoiceState, type VoiceAction } from '@/lib/voice-state';
    import { SpeechManager, isSpeechRecognitionSupported } from '@/lib/speech-recognition';
    import { TTSClient } from '@/lib/tts-client';
    import { startThinkingChime, stopThinkingChime } from '@/lib/thinking-chime';
    ```

    The hook signature:
    ```typescript
    export function useVoiceLoop(): {
      voiceState: VoiceState;
      startListening: () => void;
      stopListening: () => void;
      handleTextInput: (text: string) => void;
      isSupported: boolean;
      needsPermissionPrompt: boolean;
      dismissPermissionPrompt: () => void;
      transcript: string;
      response: string;
    }
    ```

    Implementation details:

    1. **State**: `useReducer(voiceReducer, initialVoiceState)`. Track `transcript` and `response` as derived from voiceState for easy consumption.

    2. **Refs**: `useRef` for SpeechManager and TTSClient instances. Create them lazily (not in constructor) to avoid SSR issues. Also `useRef<boolean>` for `permissionDismissed`.

    3. **Permission prompt**: `useState(true)` for `needsPermissionPrompt`. Set to false when `dismissPermissionPrompt()` called or when speech recognition starts successfully (first `onstart` event).

    4. **SpeechManager creation** (lazy, in startListening):
       - `onTranscript`: dispatch `{ type: 'SPEECH_RESULT', transcript }`. Then simulate a voice response -- for Phase 2 only, since Claude conversation is Phase 3, call TTSClient with a placeholder response: `"I heard you say: ${transcript}. Menu conversation will be available in the next update."`. In Phase 3 this will be replaced with actual Claude API streaming.
       - `onError`: dispatch `{ type: 'ERROR', message }`.
       - `onRestart`: dispatch `{ type: 'START_LISTENING' }` (the auto-restart after speaking ends).

    5. **TTSClient creation** (lazy, alongside SpeechManager):
       - `onSpeakingStart`: dispatch `{ type: 'FIRST_AUDIO_READY', response: currentResponse }`.
       - `onSpeakingEnd`: dispatch `{ type: 'PLAYBACK_ENDED' }`. Per UI-SPEC: speaking -> listening auto-restart.
       - `onSentenceStart`: update response accumulator for TranscriptDisplay.

    6. **Thinking chime effect**: `useEffect` watching `voiceState.status`:
       - If status is `'processing'`: call `startThinkingChime()`.
       - If status is NOT `'processing'`: call `stopThinkingChime()`.
       - Cleanup: `stopThinkingChime()`.

    7. **startListening()**: Create SpeechManager and TTSClient if not yet created. Dispatch `START_LISTENING`. Call `speechManager.start()`. Set `needsPermissionPrompt = false`.

    8. **stopListening()**: Dispatch `STOP`. Call `speechManager.stop()`. Call `ttsClient.stop()`. Call `stopThinkingChime()`.

    9. **handleTextInput(text)**: Same as transcript path but bypasses SpeechManager. Dispatch `SPEECH_RESULT` with text. Trigger TTS response (same placeholder as #4).

    10. **Cleanup**: `useEffect` cleanup on unmount -- destroy SpeechManager and TTSClient refs, stop chime.

    11. **Button tap handler** (exposed as part of the hook or derived by consumer): The page will map mic button taps based on current status per UI-SPEC State Machine Interaction Contract:
        - idle -> startListening()
        - listening -> stopListening()
        - processing -> no-op (button disabled)
        - speaking -> stopListening() (interrupt)
        - error -> startListening() (retry, which also dispatches RETRY internally or the page dispatches RETRY first)

    Create tests at `src/hooks/__tests__/useVoiceLoop.test.ts`:
    - Test: hook returns initial idle state
    - Test: startListening dispatches START_LISTENING (mock SpeechManager)
    - Test: stopListening dispatches STOP
    - Test: handleTextInput dispatches SPEECH_RESULT
    - Test: isSupported returns true when SpeechRecognition mock is present
    - Test: thinking chime starts on processing state, stops on non-processing state
    - Use `renderHook` from `@testing-library/react` and `act` for state transitions.
