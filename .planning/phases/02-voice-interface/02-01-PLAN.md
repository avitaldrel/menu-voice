    Create `src/lib/voice-state.ts` with the following exact types and reducer (per VOICE-04 and CONTEXT.md state machine):

    ```typescript
    export type VoiceState =
      | { status: 'idle' }
      | { status: 'listening'; transcript: string }
      | { status: 'processing'; transcript: string }
      | { status: 'speaking'; transcript: string; response: string }
      | { status: 'error'; message: string };

    export type VoiceAction =
      | { type: 'START_LISTENING' }
      | { type: 'SPEECH_RESULT'; transcript: string }
      | { type: 'FIRST_AUDIO_READY'; response: string }
      | { type: 'PLAYBACK_ENDED' }
      | { type: 'STOP' }
      | { type: 'ERROR'; message: string }
      | { type: 'RETRY' };

    export const initialVoiceState: VoiceState = { status: 'idle' };
    ```

    The `voiceReducer` must enforce these allowed transitions from UI-SPEC State Machine Interaction Contract:
    - idle -> listening (START_LISTENING)
    - listening -> processing (SPEECH_RESULT with transcript)
    - listening -> idle (STOP, or timeout handled at hook level)
    - listening -> error (ERROR)
    - processing -> speaking (FIRST_AUDIO_READY with response)
    - processing -> error (ERROR)
    - speaking -> listening (PLAYBACK_ENDED -- auto-restart)
    - speaking -> idle (STOP -- user interrupt)
    - error -> idle (RETRY)

    Any action not matching an allowed transition for the current status returns the current state unchanged.

    Create tests in `src/lib/__tests__/voice-state.test.ts` covering every valid transition AND at least 3 invalid transition cases. Follow RED-GREEN: write tests first, run to see them fail, then implement reducer.
