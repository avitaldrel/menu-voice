       export class SpeechManager {
         private recognition: SpeechRecognition | null;

         constructor(
           private onTranscript: (transcript: string) => void,

    Create `src/lib/tts-client.ts` with:

    1. `splitSentences(text: string): string[]` -- split on terminal punctuation (`.`, `?`, `!`) followed by whitespace or end-of-string. Use regex: `text.match(/[^.!?]+[.!?]+(\s|$)?/g)?.map(s => s.trim()) ?? [text.trim()]`. Filter out empty strings.

    2. `TTSClient` class:
       ```typescript
       export class TTSClient {
         private audioElement: HTMLAudioElement;
         private queue: string[] = [];
         private isPlaying: boolean = false;
         private useFallback: boolean = false;
         private requestsSinceFallback: number = 0;
         private onSpeakingStart: (() => void) | null = null;
         private onSpeakingEnd: (() => void) | null = null;
         private onSentenceStart: ((sentence: string) => void) | null = null;

         constructor(options: {
           onSpeakingStart?: () => void;
           onSpeakingEnd?: () => void;
           onSentenceStart?: (sentence: string) => void;
         });

         queueText(text: string): void;  // buffer + enqueue complete sentences
         flush(): void;                   // flush remaining buffer
         stop(): void;                    // stop playback + clear queue
         destroy(): void;                 // cleanup audio element
       }
       ```

       `queueText(text: string)`: Append text to internal buffer string. Run `splitSentences(buffer)`. If result has > 1 item, all but the last are complete sentences -- push them to the queue. Keep the last fragment as the new buffer. If not already playing, call `playNext()`.

       `flush()`: If buffer has content, push it to queue and clear buffer. If not playing, call `playNext()`.

       Private `playNext()`: If queue is empty, set `isPlaying = false`, call `onSpeakingEnd?.()`, return. Pop first sentence from queue. Call `onSentenceStart?.(sentence)`. Set `isPlaying = true`.

       Check if should try OpenAI: `!useFallback || (requestsSinceFallback % 5 === 0)`. Per CONTEXT.md: "Auto-detect fallback on first OpenAI TTS failure, switch silently to browser SpeechSynthesis, retry OpenAI every 5 requests."

       If trying OpenAI:
       - `fetch('/api/tts', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ text: sentence }) })`
       - If response not ok, set `useFallback = true`, fall through to fallback path
       - If ok: `const blob = await response.blob()`, `const url = URL.createObjectURL(blob)`, set `audioElement.src = url`, call `audioElement.play()`
       - On `audioElement.onended`: call `URL.revokeObjectURL(url)` (Pitfall 4 -- memory leak prevention), then call `playNext()`

       If using fallback (VOICE-03):
       - Use `speakWithSynthesis(sentence, () => playNext())` helper
       - `speakWithSynthesis`: create `SpeechSynthesisUtterance(text)`, set `lang: 'en-US'`, `rate: 1.0`, `volume: 1.0`
       - Store reference on window to prevent iOS GC bug (per RESEARCH SpeechSynthesis Fallback example)
       - `utterance.onend` and `utterance.onerror` both call the callback and clean up the reference
       - Call `window.speechSynthesis.speak(utterance)`

       Increment `requestsSinceFallback` after each request when in fallback mode.

       `stop()`: pause audioElement, revoke current blob URL if exists, call `window.speechSynthesis?.cancel()`, clear queue, set `isPlaying = false`.

       The `audioElement` is created in constructor via `new Audio()` -- NOT attached to DOM (the Audio() constructor creates a standalone element per A11Y-05).

       On first `speakSentence` call, also call `onSpeakingStart?.()`.

    Tests: mock fetch, test sentence splitting, test queue management, test fallback trigger on fetch failure, test blob URL revocation, test stop clears queue.
